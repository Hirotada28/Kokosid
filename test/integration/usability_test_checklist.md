# ユーザビリティテストの準備チェックリスト

このドキュメントは、Kokosidアプリケーションのユーザビリティテストを実施するための準備項目をまとめたものです。

**要件: 8.1, 8.2, 8.3, 8.4**

## 1. オンボーディングフローの最適化

### 1.1 初回起動時の価値提案説明

#### 実装項目
- [ ] スプラッシュスクリーン後の価値提案画面
  - アプリの核心的価値（心理的ミラーリング、マイクロ・チャンキング、デジタル・バディ体験）
  - 3つの主要機能の簡潔な説明
  - 「次へ」ボタンと「スキップ」オプション

#### テスト項目
- [ ] 価値提案が5秒以内に理解できるか
- [ ] 視覚的要素（アイコン、イラスト）が適切か
- [ ] テキストが読みやすいか（フォントサイズ、行間）
- [ ] 「次へ」ボタンが見つけやすいか

#### 成功基準
- ユーザーの80%以上が価値提案を理解できる
- 平均読了時間が10秒以内
- スキップ率が30%以下

### 1.2 暗号化とプライバシーの説明

#### 実装項目
- [ ] プライバシー保護の説明画面
  - エンドツーエンド暗号化の仕組み
  - データがデバイス内で暗号化されること
  - サーバー側でキーを持たないこと
  - ユーザーのデータは完全にプライベートであること

#### テスト項目
- [ ] 暗号化の説明が専門用語なしで理解できるか
- [ ] プライバシー保護の重要性が伝わるか
- [ ] 不安を感じさせない表現になっているか
- [ ] 「同意する」ボタンが明確か

#### 成功基準
- ユーザーの90%以上がプライバシー保護を理解できる
- 不安を感じるユーザーが10%以下
- 同意率が95%以上

### 1.3 基本操作のチュートリアル

#### 実装項目
- [ ] インタラクティブチュートリアル
  - タスク作成の方法
  - 音声入力の使い方
  - 3タブナビゲーションの説明
  - 初回タスク作成のガイド

#### テスト項目
- [ ] チュートリアルが5分以内に完了できるか
- [ ] 各ステップが明確か
- [ ] スキップオプションが適切に配置されているか
- [ ] 実際の操作を試せるか

#### 成功基準
- ユーザーの85%以上がチュートリアルを完了できる
- 平均完了時間が3分以内
- 完了後、基本操作を理解しているユーザーが90%以上

## 2. アクセシビリティ対応

### 2.1 スクリーンリーダー対応

#### 実装項目
- [ ] 全てのUI要素にセマンティックラベルを設定
  - ボタン、テキストフィールド、画像
  - ナビゲーション要素
  - 動的コンテンツ

- [ ] 適切な読み上げ順序の設定
  - 論理的な順序でコンテンツが読み上げられる
  - 重要な情報が優先される

- [ ] 状態変化の通知
  - タスク完了時の音声フィードバック
  - エラーメッセージの読み上げ
  - 進捗状況の通知

#### テスト項目
- [ ] TalkBack（Android）での動作確認
- [ ] VoiceOver（iOS）での動作確認
- [ ] 全ての画面が音声のみで操作できるか
- [ ] 読み上げ内容が適切か

#### 成功基準
- 全てのUI要素が適切に読み上げられる
- スクリーンリーダーユーザーが全機能を使用できる
- WCAG 2.1 レベルAA準拠

### 2.2 色覚多様性への配慮

#### 実装項目
- [ ] 色だけに依存しない情報伝達
  - アイコンとテキストの併用
  - パターンや形状の活用
  - 十分なコントラスト比の確保

- [ ] カラーパレットの最適化
  - 色覚異常シミュレーションでの検証
  - 赤緑色覚異常に配慮した配色
  - 青黄色覚異常に配慮した配色

#### テスト項目
- [ ] 色覚異常シミュレーターでの確認
  - 1型色覚（赤色盲）
  - 2型色覚（緑色盲）
  - 3型色覚（青色盲）

- [ ] コントラスト比の測定
  - テキストと背景: 最低4.5:1
  - 大きなテキスト: 最低3:1
  - UI要素: 最低3:1

#### 成功基準
- 全ての色覚タイプで情報が識別できる
- WCAG 2.1 コントラスト要件を満たす
- 色覚異常ユーザーからのフィードバックが良好

### 2.3 フォントサイズ調整機能

#### 実装項目
- [ ] システムフォントサイズの尊重
  - OSのフォントサイズ設定に従う
  - 動的なレイアウト調整

- [ ] アプリ内フォントサイズ設定
  - 小・中・大・特大の4段階
  - リアルタイムプレビュー
  - 設定の永続化

#### テスト項目
- [ ] 最小フォントサイズでの可読性
- [ ] 最大フォントサイズでのレイアウト崩れ
- [ ] 各画面での動作確認
- [ ] 設定変更の即時反映

#### 成功基準
- 全てのフォントサイズで情報が読める
- レイアウトが崩れない
- ユーザーの95%以上が快適に読める

## 4. ユーザビリティテストの実施計画

### 4.1 テスト参加者

#### 募集基準
- [ ] 多様な背景を持つ参加者
  - 年齢: 20代〜50代
  - 性別: 男女
  - 技術リテラシー: 初心者〜上級者
  - ADHD傾向: あり・なし

- [ ] 参加者数
  - 最低10名（各グループ2名以上）
  - アクセシビリティテスト: 追加5名

### 4.2 テストシナリオ

#### シナリオ1: 初回起動とオンボーディング
1. アプリを初めて起動する
2. オンボーディングを完了する
3. 初回タスクを作成する

#### シナリオ2: タスク管理フロー
1. タスクを作成する
2. マイクロ分解を確認する
3. タスクを完了する
4. お祝い演出を体験する

#### シナリオ3: 音声入力と対話
1. 音声で気持ちを記録する
2. AI対話を体験する
3. 感情分析結果を確認する

#### シナリオ4: 自己肯定感の確認
1. 「軌跡」タブを開く
2. 自己肯定感スコアを確認する
3. 算出根拠を理解する

### 4.3 評価指標

#### 定量的指標
- [ ] タスク完了率
- [ ] タスク完了時間
- [ ] エラー発生率
- [ ] ヘルプ参照回数

#### 定性的指標
- [ ] System Usability Scale (SUS)
- [ ] ユーザー満足度（5段階評価）
- [ ] 推奨度（NPS）
- [ ] 自由記述フィードバック

### 4.4 成功基準

#### 全体目標
- [ ] SUSスコア: 70以上（良好）
- [ ] タスク完了率: 85%以上
- [ ] ユーザー満足度: 4.0以上（5段階）
- [ ] NPS: 30以上

#### 個別目標
- [ ] オンボーディング完了率: 90%以上
- [ ] 初回タスク作成成功率: 85%以上
